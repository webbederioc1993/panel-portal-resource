# Panel Portal Resource

## 项目简介

`panel-portal-resource` 是一个用于归档与发布独立 HTML 页面的资源仓库。本仓库旨在收集、整理并提供一系列自包含的 Web 页面，涵盖各类工具、面板、信息展示页等场景。所有页面均为静态 HTML，不依赖外部动态服务，可直接在浏览器中打开使用。

本仓库不针对任何特定域名或网站，仅作为页面资源的集中存储与版本管理用途。使用者可根据自身需求，自由引用或部署其中的页面。

## 目录结构

```
panel-portal-resource/
├── pages/               # 存放独立的 HTML 页面文件
│   ├── page-a.html      # 示例页面 A
│   ├── page-b.html      # 示例页面 B
│   └── ...
├── assets/              # 页面共用资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
├── docs/                # 相关文档或说明
│   └── archive-note.md
└── README.md            # 本文件
```

- `pages/`：主要页面归档目录，每个文件为一个完整的独立页面。
- `assets/`：页面可能引用的公共资源，建议页面尽量内联或自包含，以保持独立性。
- `docs/`：存放归档说明、更新日志或其他附加文档。

## 页面归档说明

- 所有页面均为纯 HTML，可独立运行，无需后端支持。
- 页面内容可能包含演示、工具、信息面板等，不限定具体用途。
- 每个页面在提交时需保持完整，包括必要的样式与脚本（可内联或引用 assets 目录下的资源）。
- 建议页面文件名简洁明了，反映页面功能或主题。

### 提交新页面

1. 将页面文件放入 `pages/` 目录。
2. 如有共用资源，放入 `assets/` 对应子目录。
3. 在提交信息中简要说明页面用途或变更内容。
4. 确保页面无外部依赖或已明确声明依赖关系。

## 维护说明

- 本仓库由维护者不定期更新，接受 Issue 和 Pull Request。
- 欢迎提交新的独立页面或改进现有页面，但请确保页面内容合规、无恶意代码。
- 页面内容应保持中立、通用，避免包含特定宣传、推广或敏感信息。
- 对于长期未更新或存在兼容问题的页面，维护者可能进行清理或标注。

### 贡献指南

- Fork 本仓库，创建新分支。
- 提交修改后，发起 Pull Request，等待审核。
- 请确保页面在不同浏览器（Chrome、Firefox、Edge 等）下基本可用。
- 如有必要，可在 `docs/` 中添加对应说明文档。
